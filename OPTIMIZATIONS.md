# Оптимизации проекта

## Примененные оптимизации

### 1. Кэширование изображений
- **Пакет**: `cached_network_image` + `flutter_cache_manager`
- **Что делает**: Автоматически кэширует все загруженные изображения на устройстве
- **Преимущества**:
  - Изображения загружаются только один раз
  - Мгновенная загрузка при повторном просмотре
  - Работает офлайн с кэшированными изображениями
  - Снижает потребление трафика
  - Уменьшает нагрузку на сервер

### 2. Виджет CachedImage
- **Расположение**: `lib/widgets/cached_image.dart`
- **Функции**:
  - Автоматическое кэширование
  - Плавная анимация появления (fade in/out)
  - Красивый placeholder с индикатором загрузки
  - Обработка ошибок
  - Настраиваемое скругление углов

### 3. Утилиты производительности
- **Расположение**: `lib/utils/performance_utils.dart`
- **Включает**:
  - **Debouncer**: Защита от частых вызовов (поиск, фильтрация)
  - **Throttler**: Ограничение частоты вызовов (скролл, анимации)
  - **Memoizer**: Кэширование результатов функций

### 4. Оптимизация списков
- Использование `SliverList` вместо `ListView` для больших списков
- Ленивая загрузка элементов
- Правильное использование ключей (`ValueKey`)

### 5. Оптимизация виджетов
- Использование `const` конструкторов где возможно
- `StatelessWidget` вместо `StatefulWidget` где нет состояния
- Правильное использование `context.watch` vs `context.read`

## Как использовать

### CachedImage вместо Image.network
```dart
// Было:
Image.network(
  product.images[i],
  fit: BoxFit.cover,
)

// Стало:
CachedImage(
  imageUrl: product.images[i],
  fit: BoxFit.cover,
  borderRadius: 12,
)
```

### Debouncer для поиска
```dart
final _debouncer = Debouncer(delay: Duration(milliseconds: 500));

// В onChanged текстового поля:
onChanged: (value) {
  _debouncer(() {
    // Выполнить поиск только после прекращения ввода
    performSearch(value);
  });
}
```

### Throttler для скролла
```dart
final _throttler = Throttler(interval: Duration(milliseconds: 300));

// В слушателе скролла:
_scrollController.addListener(() {
  _throttler(() {
    // Обработать скролл не чаще 3 раз в секунду
    handleScroll();
  });
});
```

## Рекомендации по дальнейшей оптимизации

### 1. Пагинация
- Добавить подгрузку товаров порциями (по 20-50 штук)
- Использовать `PagingController` из пакета `infinite_scroll_pagination`

### 2. Оптимизация Firestore запросов
- Индексировать часто используемые поля
- Использовать `.limit()` для ограничения результатов
- Кэшировать данные локально с помощью `shared_preferences`

### 3. Lazy loading для тяжелых компонентов
- Видеоплеер загружать только при открытии
- Аудиоплеер инициализировать по требованию
- Использовать `FutureBuilder` и `StreamBuilder` правильно

### 4. Tree shaking и code splitting
- Удалять неиспользуемые импорты
- Разбивать большие файлы на меньшие
- Использовать `deferred` imports для редко используемых модулей

### 5. Оптимизация изображений на сервере
- Использовать WebP формат вместо PNG/JPG
- Создавать миниатюры разных размеров
- Использовать CDN для доставки изображений

### 6. Профилирование
- Использовать Flutter DevTools для анализа производительности
- Проверять memory leaks
- Оптимизировать rebuild виджетов

## Заметки

- Кэш изображений автоматически очищается при нехватке места
- Максимальный размер кэша: 500 MB (по умолчанию)
- Время жизни кэша: 30 дней (по умолчанию)
- Можно настроить в `flutter_cache_manager`
